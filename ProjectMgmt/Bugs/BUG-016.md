## Summary

The CRC checker module existed but was not fully integrated with the data path. CRC errors were not properly detected or reported.

---

## Problem Description

### Module Interface
```verilog
module crc_checker (
    input         clk,
    input         rst_n,
    input         data_valid,
    input  [31:0] data_in,
    input         crc_en,
    output [31:0] crc_out,
    output        crc_error
);
```

### Original Implementation (Bug)
```verilog
// aes_top.v (Bug)
// CRC checker instantiated but not connected
crc_checker u_crc (
    .clk(clk),
    .rst_n(rst_n),
    .data_valid(1'b0),  // Not connected!
    .data_in(32'h0),    // Not connected!
    .crc_en(1'b0),      // Not connected!
    .crc_out(),
    .crc_error()        // Not used!
);
```

**问题：** CRC检查器存在但没有连接到数据路径，无法检测错误。

### Fixed Implementation
```verilog
// aes_top.v (Fixed)
// CRC checker fully integrated
crc_checker u_crc (
    .clk(clk),
    .rst_n(rst_n),
    .data_valid(data_out_valid),  // Connected
    .data_in(ciphertext[31:0]),   // Connected
    .crc_en(crc_check_enable),    // Controlled by mode
    .crc_out(crc_result),
    .crc_error(crc_error)         // Connected to status_reg
);

// Connect to ERR_STAT
assign status_reg[3] = crc_error;  // CRC_MISMATCH bit
```

---

## Root Cause Analysis

1. **信号未连接**: data_valid/data_in/crc_en未连接到系统信号
2. **错误输出未使用**: crc_error未连接到ERR_STAT
3. **无使能控制**: crc_en没有根据模式自动使能
4. **数据路径缺失**: 密文数据未送入CRC检查器

---

## Fix Details

### Changes Made

1. **连接数据有效信号**
   - data_valid <= data_out_valid
   - 在数据输出时触发CRC计算

2. **连接数据输入**
   - data_in <= ciphertext[31:0]
   - 逐块处理128-bit密文

3. **添加CRC使能控制**
   - crc_en由mode_controller控制
   - 特定模式下自动使能

4. **连接错误输出**
   - crc_error -> status_reg[3]
   - ERR_STAT新增CRC_MISMATCH位
