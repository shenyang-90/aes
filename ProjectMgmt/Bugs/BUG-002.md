## Summary

Testcases use unpacked structs which are not supported by Iverilog, preventing coverage collection.

## Error Details

```
./Testcases/directed/tc_ecb_nist.sv:14: sorry: Unpacked structs not supported.
```

## Affected Testcases

| Testcase | Status |
|----------|--------|
| tc_ecb_nist.sv | FAIL |
| tc_cbc_nist.sv | FAIL |
| tc_ctr_nist.sv | FAIL |
| tc_cts_boundary.sv | FAIL |
| tc_smoke.sv | PASS |

## Root Cause

Iverilog does not support unpacked structs/typedefs:
```systemverilog
typedef struct {
    string name;
    bit [127:0] key;
    ...
} vector_t;
```

## Impact

- Cannot run NIST vector tests with open-source tools
- Coverage collection blocked at ~20% pass rate
- TASK-AES-COV-001 cannot reach >90% target with iverilog

## Proposed Solutions

### Option 1: Refactor Testcases (Recommended for iverilog)
Replace structs with simple arrays:
```systemverilog
reg [127:0] keys [0:6];
reg [127:0] plaintexts [0:6];
reg [127:0] ciphertexts [0:6];
```

### Option 2: Use Commercial Simulator
Use VCS/Xcelium which supports full SystemVerilog.

## Required Actions

- [ ] Verification Agent: Re-run coverage collection
