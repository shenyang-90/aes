## Summary

CRC checker 模块中存在逻辑错误，未使用实际的 `data_in` 进行 CRC 计算，导致 CRC 检查功能失效。

---

## Problem Description

### Current Implementation (Bug)
```verilog
// crc_checker.v:55 (approximately)
// BUG: data_in not used in calculation
always @(posedge clk) begin
    if (calc_en) begin
        // Current: Using some internal value instead of data_in
        crc_reg <= next_crc(crc_reg, some_internal_signal);  // ❌ Wrong
    end
end
```

### Correct Implementation
```verilog
// crc_checker.v
// Should use data_in for calculation
always @(posedge clk) begin
    if (calc_en) begin
        crc_reg <= next_crc(crc_reg, data_in);  // ✅ Correct
    end
end
```

### Expected Behavior
CRC checker 应该：
1. 接收 `data_in` 和 `data_valid`
2. 使用标准 CRC-32 算法计算 CRC
3. 与预期的 CRC 值比较
4. 输出 `crc_ok` 或 `crc_error`

---

## Impact

- **功能安全**: ASIL-D 要求的完整性检查失效
- **故障检测**: 无法检测数据传输/存储错误
- **安全**: 潜在的故障注入攻击窗口

## Implementation Details

### Standard CRC-32
```verilog
// CRC-32 (Ethernet, ZIP, etc.)
// Polynomial: x^32 + x^26 + x^23 + x^22 + x^16 + x^12 + x^11 + 
//             x^10 + x^8 + x^7 + x^5 + x^4 + x^2 + x + 1
// Hex: 0x04C11DB7

function [31:0] crc32_byte;
    input [31:0] crc;
    input [7:0] data;
    integer i;
    reg feedback;
    reg [31:0] crc_tmp;
    begin
        crc_tmp = crc;
        for (i = 0; i < 8; i = i + 1) begin
            feedback = crc_tmp[31] ^ data[i];
            crc_tmp = {crc_tmp[30:0], 1'b0};
            if (feedback)
                crc_tmp = crc_tmp ^ 32'h04C11DB7;
        end
        crc32_byte = crc_tmp;
    end
endfunction
```

### Module Interface Check
```verilog
module crc_checker (
    input         clk,
    input         rst_n,
    input  [31:0] data_in,       // Data to check
    input         data_valid,    // Data valid

    input  [31:0] expected_crc,  // Expected CRC value
    output reg    crc_ok,        // CRC match

);
```
