## Summary

CRC checker 模块中存在逻辑错误，未使用实际的 `data_in` 进行 CRC 计算，导致 CRC 检查功能失效。

---

## Problem Description

### Original Implementation (Bug)
```verilog
// crc_checker.v:55 (approximately)
// BUG: data_in not used in calculation
CALC: begin
    if (bit_cnt < 128) begin
        // Simple CRC calculation (placeholder)
        crc_reg <= {crc_reg[30:0], 1'b0} ^ (crc_reg[31] ? CRC_POLY : 32'd0);
        bit_cnt <= bit_cnt + 1'b1;
    end else begin
        state <= DONE;
    end
end
```

**问题：** CRC 计算只使用了 `crc_reg` 自身的状态，没有将 `data_in` 的数据纳入计算。

### Fixed Implementation
```verilog
// crc_checker.v
// Fixed: Now correctly uses data_in for calculation
CALC: begin
    if (bit_cnt < 8'd128) begin
        // Get current bit from data_in (MSB first: bit 127 down to 0)
        if (crc_reg[31] ^ data_in[127 - bit_cnt[6:0]]) begin
            crc_reg <= {crc_reg[30:0], 1'b0} ^ CRC_POLY;
        end else begin
            crc_reg <= {crc_reg[30:0], 1'b0};
        end
        bit_cnt <= bit_cnt + 8'd1;
    end else begin
        state <= DONE;
    end
end
```

### Expected Behavior
CRC checker 应该：
1. 接收 `data_in` 和 `calc_start` 信号
2. 使用标准 CRC-32 算法计算 CRC (多项式 0x04C11DB7)
3. 逐位处理 data_in (MSB first)
4. 输出计算完成的 CRC 值

---

## Root Cause Analysis

1. **bit_cnt 位宽不足：** 原使用 7-bit 计数器 (0-127)，无法正确表示 128 这个计数值，导致状态机无法正常退出 CALC 状态
2. **未使用 data_in：** CRC 计算逻辑中没有将输入数据与 CRC 寄存器进行 XOR 运算

---

## Fix Details

### Changes Made

1. **将 bit_cnt 从 7-bit 改为 8-bit**
   - 原：`reg [6:0] bit_cnt;`
   - 改：`reg [7:0] bit_cnt;`
   - 原因：需要表示 0-128 的范围才能正确检测 128 位处理完成

2. **修改 CRC 计算逻辑，使用 data_in**
   - 原：`if (crc_reg[31])` (只检查 CRC MSB)
   - 改：`if (crc_reg[31] ^ data_in[127 - bit_cnt[6:0]])` (CRC MSB XOR data bit)
   - 原因：标准 CRC-32 算法需要将输入数据位与 CRC 寄存器最高位进行 XOR
