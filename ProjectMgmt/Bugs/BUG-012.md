## Summary

XTS mode multi-sector tweak calculation was incomplete. The implementation only handled single-sector encryption/decryption but did not properly calculate tweak values for subsequent sectors.

---

## Problem Description

### Original Implementation (Bug)
```verilog
// xts_engine.v (Bug)
// Only single sector supported
always @(posedge clk) begin
    if (start) begin
        // Calculate tweak for sector 0 only
        tweak <= encrypt_sector_key(sector_num);
    end
end
// No sector increment logic
```

**问题：** XTS引擎不支持多Sector操作，无法实现磁盘加密场景。

### Fixed Implementation
```verilog
// xts_engine.v (Fixed)
// Multi-sector support with sector_offset input
always @(posedge clk) begin
    if (start || sector_inc) begin
        // Calculate tweak for current sector
        tweak <= encrypt_sector_key(sector_num + sector_offset);
    end
end
// Added NEXT_SECTOR state for multi-sector operation
```

### Expected Behavior
XTS mode should:
1. Support multi-sector operations (sector 0, 1, 2, ...)
2. Calculate unique tweak for each sector: T = E_k2(sector_number)
3. Properly chain blocks within each sector
4. Support sector sizes: 512 bytes, 1024 bytes, 4096 bytes

---

## Root Cause Analysis

1. **缺少Sector偏移输入**: 无法指定起始sector号
2. **无Sector递增逻辑**: 完成一个sector后无法自动进入下一个
3. **BUG-008仅修复块内链式**: alpha^j乘法只处理了块内，未处理sector间
4. **Tweak计算单一**: 只计算了sector 0的tweak

---

## Fix Details

### Changes Made

1. **添加Sector偏移输入**
   - 新增`sector_offset`输入端口
   - 支持从任意sector开始
   - sector地址可配置

2. **添加Sector递增逻辑**
   - 新增`sector_inc`输入信号
   - 添加`NEXT_SECTOR`状态
   - 自动计算下一个sector的tweak

3. **完善Tweak计算**
   - T = AES_enc(Key2, sector_number + sector_offset)
   - 每个sector有独立的初始tweak
   - 保持BUG-008的块内链式

4. **支持多种Sector大小**
   - 可配置sector_size参数
   - 支持512B/1024B/4096B
   - 自适应块数量计算
