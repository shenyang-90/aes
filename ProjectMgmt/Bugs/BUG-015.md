## Summary

No mechanism existed to clear keys from the key_manager registers. Once a key was loaded, it persisted until reset.

---

## Problem Description

### Original Implementation (Bug)
```verilog
// key_manager.v / aes_top.v (Bug)
// Keys can only be loaded, never cleared
always @(posedge clk) begin
    if (key_load) begin
        key_reg <= key_in;  // Only load, no clear!
    end
end
// No key_clear logic
```

**问题：** 密钥一旦加载就无法清除，存在安全隐患。

### Fixed Implementation
```verilog
// aes_top.v (Fixed)
// Key clear via CTRL[9] bit
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) 
        key_reg <= 256'h0;
    else if (key_clear)  // CTRL[9] = 1
        key_reg <= 256'h0;  // Secure zeroization
    else if (key_load)
        key_reg <= key_in;
end
```

### Expected Behavior
Key manager should support:
1. Key clear command (software-initiated)
2. Automatic key clear on error (可选)
3. Key retention across operations (可选)
4. Secure key zeroization (全0覆盖)

---

## Root Cause Analysis

1. **缺少清除控制**: CTRL寄存器没有key_clear位
2. **无安全清零**: 密钥无法被显式覆盖
3. **依赖复位**: 只能通过硬件复位清除
4. **不符合安全规范**: ASIL-D要求密钥可清除

---

## Fix Details

### Changes Made

1. **添加CTRL[9] Key Clear位**
   - 写1清除所有密钥寄存器
   - 写0无操作
   - 自清除或持续有效(选择自清除)

2. **实现安全清零**
   - 256位全部清零
   - key_reg[255:0] = 0
   - 覆盖所有密钥长度(128/192/256)

3. **添加key_valid状态**
   - 清除后key_valid = 0
   - 加载后key_valid = 1
   - 用于后续操作检查

4. **保持key_schedule同步**
   - 清除时同时清除schedule输出
   - 防止残留密钥信息
