## Summary

状态机命名与 Design Specification 文档不一致，可能导致维护困难和理解偏差。

---

## Problem Description

### 文档定义
根据 `Database/Docs/Design/Design_Specification.md` Section 6.2：
```
主控制状态机:
IDLE → KEY_SCHEDULE → LOAD_DATA → ROUND_OP → FINAL_ROUND → OUTPUT → DONE
```

### RTL 实现
实际代码中的状态命名：
```verilog
// aes_controller.v
IDLE → LOAD_KEY → WAIT_KEY → LOAD_IV → WAIT_DATA → PROCESS → WAIT_CORE → OUTPUT → DONE
```

### Differences

| 文档状态 | RTL 状态 | 说明 |
|---------|---------|------|
| KEY_SCHEDULE | LOAD_KEY + WAIT_KEY | 拆分为两个状态 |
| LOAD_DATA | LOAD_IV + WAIT_DATA | 包含 IV 加载 |
| ROUND_OP | PROCESS + WAIT_CORE | 拆分为执行和等待 |
| FINAL_ROUND | (在 PROCESS 中处理) | 合并到处理状态 |

---

## Impact

- **维护**: 新开发人员阅读文档和代码时会有困惑
- **验证**: 验证计划中的状态机检查可能需要调整
- **功能**: 逻辑正确，仅命名不一致
