## Summary

UVM testbench compilation failed. iverilog does not support UVM SystemVerilog constructs.

## Error Details

```
./tb_top.sv:10: syntax error
./tb_top.sv:10: error: invalid module item.
./tb_top.sv:11: syntax error
./tb_top.sv:84: syntax error
./tb_top.sv:85: error: malformed statement
```

## Root Cause

The UVM verification environment uses advanced SystemVerilog features:
- `import uvm_pkg::*`
- UVM macros (`uvm_component_utils`, etc.)
- Interface with modports
- UVM sequencer/driver/monitor classes

These features are not supported by open-source simulators like Iverilog or Verilator.

## Solution Implemented

Created simplified testbench (`tb_simple.sv`) with:
- Basic SystemVerilog compatible with iverilog
- Direct APB/AXI transaction tasks
- Self-checking test sequence
- VCD waveform dump
