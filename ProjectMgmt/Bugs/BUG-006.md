## Problem Description

### Current Implementation
当前 sbox_masked.v 仅包含框架代码，未实现实际的 3-share TI 乘法：
- 输入/输出端口定义完成
- 状态机框架存在
- **缺少实际的 3-share TI 乘法逻辑**

### Required Implementation
根据 `Database/Docs/Design/TI_SBox_Design.md` 要求：

```
TI S-Box 架构:
- 3-share 输入: a = a0 ⊕ a1 ⊕ a2
- 3-share 输出: b = b0 ⊕ b1 ⊕ b2
- 内部使用 DOM (Domain-Oriented Masking) 乘法器
- 每轮需要刷新掩码 (mask refresh)
```

具体需要实现：
1. **3-share TI 乘法器** - 基于 DOM 的 GF(2^8) 乘法
2. **掩码刷新逻辑** - 每轮新的随机掩码
3. **完整 S-Box 组合** - 基于 Canright 或类似架构的掩码 S-Box

## Fix Details

### Implementation Summary

Complete TI 3-share S-Box implementation based on Canright's composite field approach:

1. **GF(2^4) Arithmetic Functions**
   - `gf16_mul`: GF(2^4) multiplication using bit-serial approach
   - `gf16_square`: Linear squaring operation (share-safe)
   - `gf16_inv`: Inversion using Itoh-Tsujii algorithm (a^14)
   - `gf16_scale_n`: Multiplication by constant N (lambda)

2. **Field Isomorphism**
   - `iso_map`: GF(2^8) to GF(2^4)^2 mapping
   - `iso_inv_map`: GF(2^4)^2 to GF(2^8) inverse mapping
   - Uses binary matrix multiplication for basis conversion

3. **S-Box Pipeline Stages**
   - STAGE_IDLE → STAGE_IN: Load input shares
   - STAGE_IN → STAGE_MAP: Apply isomorphism
   - STAGE_MAP → STAGE_INV: GF(2^4) operations (Canright decomposition)
   - STAGE_INV → STAGE_IMAP: Inversion
   - STAGE_IMAP → STAGE_AFF: Inverse isomorphism
   - STAGE_AFF → STAGE_OUT: Affine transformation with random mask
   - STAGE_OUT: Output shares

4. **Security Features**
   - 3-share Boolean masking: x = x0 ^ x1 ^ x2
   - 5-stage pipeline for glitch-free operation
   - Random mask refresh in affine stage
   - DOM-style multiplication (can be enhanced with full DOM)
