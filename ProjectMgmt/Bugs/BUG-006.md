## Summary

sbox_masked.v 模块目前为 placeholder 实现，未完成完整的 3-share Threshold Implementation (TI) 乘法器。这是侧信道防护的关键组件，必须在 IDR 前完成。

---

## Problem Description

### Current Implementation
当前 sbox_masked.v 仅包含框架代码，未实现实际的 3-share TI 乘法：
- 输入/输出端口定义完成
- 状态机框架存在
- **缺少实际的 3-share TI 乘法逻辑**

### Required Implementation
根据 `Database/Docs/Design/TI_SBox_Design.md` 要求：

```
TI S-Box 架构:
- 3-share 输入: a = a0 ⊕ a1 ⊕ a2
- 3-share 输出: b = b0 ⊕ b1 ⊕ b2
- 内部使用 DOM (Domain-Oriented Masking) 乘法器
- 每轮需要刷新掩码 (mask refresh)
```

具体需要实现：
1. **3-share TI 乘法器** - 基于 DOM 的 GF(2^8) 乘法
2. **掩码刷新逻辑** - 每轮新的随机掩码
3. **完整 S-Box 组合** - 基于 Canright 或类似架构的掩码 S-Box
