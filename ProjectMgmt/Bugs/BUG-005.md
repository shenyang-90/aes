## Summary

XTS-AES mode sector tweak implementation had issues. Different sector IDs produced same ciphertext output, indicating tweak value was not being correctly applied.

**Current Status:** 🟡 **6/7 tests passing** - 1 test still failing (round-trip)

### Root Cause Analysis

1. **Tweak Not Applied**: `xts_engine.v` may not be receiving sector ID from controller
2. **Tweak Calculation**: Tweak = E(Key2, SectorID) not implemented correctly
3. **XTS Chaining**: Tweak update (multiply by α in GF(2^128)) may be missing

## Expected Behavior (IEEE P1619)

```
XTS-AES Encryption:
T = E(Key2, SectorID)           // Initial tweak
for i = 0 to m-1:
    C[i] = E(Key1, P[i] ⊕ T) ⊕ T  // Encrypt with tweak
    T = T ⊗ α (in GF(2^128))    // Update tweak for next block
```

Where:
- Key1: Data encryption key (upper 128/256 bits)
- Key2: Tweak encryption key (lower 128 bits)
- α = x (polynomial representation) = 0x02 in GF(2^128)

### 3. GF(2^128) Multiplication by α (IMPLEMENTED ✅)
```verilog
// Multiply by α (x) in GF(2^128)
function [127:0] gf_mul_alpha;
    input [127:0] a;
    reg [127:0] result;
    begin
        result = {a[126:0], 1'b0};  // Shift left (multiply by x)
        if (a[127])  // If MSB was 1, reduce
            result = result ^ 128'h87;  // Reduction polynomial
        gf_mul_alpha = result;
    end
endfunction
```
