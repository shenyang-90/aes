**Status:** Open

**Assignee:** Design Agent (RTL Developer)

## Summary

XTS-AES mode sector tweak implementation has issues. Different sector IDs produce same ciphertext output, indicating tweak value is not being correctly applied.

## Error Details

### Test Failure Log
```
--- Test 3: Sector Boundary Handling ---
Sector 0 CT: 00000000000000000000000000000001
Sector 1 CT: 00000000000000000000000000000001  (SAME!)
[WARN] Different sectors produce same ciphertext

--- Test 5: Multi-sector Sequential Processing ---
Sector 0: CT=aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaab
Sector 1: CT=aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaab  (SAME!)
Sector 2: CT=aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaab  (SAME!)
[WARN] Some sectors produced same output
```

### Root Cause Analysis

1. **Tweak Not Applied**: `xts_engine.v` may not be receiving sector ID from controller
2. **Tweak Calculation**: Tweak = E(Key2, SectorID) not implemented correctly
3. **XTS Chaining**: Tweak update (multiply by α in GF(2^128)) may be missing

## Expected Behavior (IEEE P1619)

```
XTS-AES Encryption:
T = E(Key2, SectorID)           // Initial tweak
for i = 0 to m-1:
    C[i] = E(Key1, P[i] ⊕ T) ⊕ T  // Encrypt with tweak
    T = T ⊗ α (in GF(2^128))    // Update tweak for next block
```

Where:
- Key1: Data encryption key (upper 128/256 bits)
- Key2: Tweak encryption key (lower 128 bits)
- α = x (polynomial representation) = 0x02 in GF(2^128)

## Current Implementation Check

Please verify in `xts_engine.v`:

```verilog
// Should have:
module xts_engine (
    input [255:0] xts_key,      // {Key1[255:128], Key2[127:0]}
    input [127:0] sector_id,    // From IV register
    input [127:0] plaintext,
    output [127:0] ciphertext,
    // ...
);
    // Tweak generation
    wire [127:0] initial_tweak = aes_encrypt(key2, sector_id);
    
    // Tweak update per block
    wire [127:0] next_tweak = gf_mul_alpha(current_tweak);
    
    // XTS encryption: C = AES(P ⊕ T) ⊕ T
    assign ciphertext = aes_result ^ tweak;
endmodule
```

## Required Fixes

### 1. xts_engine.v
- [ ] Verify sector_id is connected from controller
- [ ] Implement tweak generation: T = E(Key2, SectorID)
- [ ] Implement GF(2^128) multiplication by α for tweak update
- [ ] Apply tweak: P ⊕ T before AES, C ⊕ T after AES

### 2. mode_controller.v
- [ ] Pass sector_id (IV) to xts_engine correctly
- [ ] Handle XTS-specific key loading (Key1 + Key2)

### 3. GF(2^128) Multiplication by α
```verilog
// Multiply by α (x) in GF(2^128)
function [127:0] gf_mul_alpha;
    input [127:0] a;
    reg [127:0] result;
    begin
        result = {a[126:0], 1'b0};  // Shift left (multiply by x)
        if (a[127])  // If MSB was 1, reduce
            result = result ^ 128'h87;  // Reduction polynomial
        gf_mul_alpha = result;
    end
endfunction
```
