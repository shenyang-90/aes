## Summary

CTS (Ciphertext Stealing) mode decryption was not implemented. The cts_handler module only supported encryption path.

---

## Problem Description

### Original Implementation (Bug)
```verilog
// cts_handler.v (Bug)
// Only encryption implemented
module cts_handler (
    input encrypt,  // Ignored - always encrypts
    // ...
);
// No decrypt state machine
always @(posedge clk) begin
    case (state)
        ENC_START: // ...
        ENC_PROCESS: // ...
        ENC_FINAL: // ...
        // No DECRYPT states!
    endcase
end
```

**问题：** CTS处理器只能加密不能解密，限制了使用场景。

### Fixed Implementation
```verilog
// cts_handler.v (Fixed)
// Both encrypt and decrypt paths
always @(posedge clk) begin
    case (state)
        // Encryption states
        ENC_START, ENC_PROCESS, ENC_FINAL: // ...
        // Decryption states (NEW)
        DEC_GET_PARTIAL, DEC_DECRYPT_2, DEC_RECONSTRUCT: // ...
    endcase
end
```

### Expected Behavior
CTS mode should support:
1. Encryption: last two blocks swapped (ciphertext stealing)
2. Decryption: reverse process to recover original plaintext
3. Support 1-127 bit partial final block

---

## Root Cause Analysis

1. **解密状态机缺失**: 只有加密状态，没有解密状态
2. **反向算法未实现**: Ciphertext stealing的逆过程未编写
3. **块重构逻辑缺失**: 无法从CTS格式恢复原始块
4. **部分块处理不完整**: 1-127 bit边界处理只实现了加密方向

---

## Fix Details

### Changes Made

1. **添加解密状态机**
   - `DEC_GET_PARTIAL`: 获取部分块
   - `DEC_DECRYPT_2`: 解密第二个块
   - `DEC_RECONSTRUCT`: 重构原始块
   - `DEC_OUTPUT`: 输出解密结果

2. **实现反向CTS算法**
   - 解密C_{n-1}得到D_n
   - 取前'length'字节作为P_n
   - 用C_n填充D_n得到C'_{n-1}
   - 解密C'_{n-1}得到P_{n-1}

3. **完善encrypt方向处理**
   - 根据encrypt信号选择路径
   - 加密路径保持原有功能
   - 解密路径新增完整实现

4. **验证round-trip**
   - 加密后解密恢复原始数据
   - 所有1-127 bit边界测试
   - 与tc_cts_boundary兼容
