## Summary

XTS 引擎中的 tweak 计算对 block_num 的处理进行了简化，未实现完整的 alpha^block_num 计算循环。这可能导致多扇区/多块处理的 tweak 值不正确。

---

## Problem Description

### Current Implementation (Simplified)
```verilog
// xts_engine.v:101 (approximately)
// Current simplified approach
assign tweak = base_tweak;  // 简化处理，未考虑 block_num
```

### Required Implementation
根据 IEEE P1619 和 `Database/Docs/Design/CTS_XTS_Design.md`：

```verilog
// Correct XTS tweak calculation
T = E(Key2, SectorID)           // Initial tweak for block 0
for i = 0 to block_num-1:
    T = T ⊗ α (in GF(2^128))    // Multiply by alpha for each block
```

每个块需要不同的 tweak：
- Block 0: T₀ = E(Key2, SectorID)
- Block 1: T₁ = T₀ ⊗ α
- Block 2: T₂ = T₁ ⊗ α = T₀ ⊗ α²
- Block n: Tₙ = T₀ ⊗ αⁿ

### Issue Location
文件: `xts_engine.v`  
行号: 约 101 行（tweak 计算逻辑）

## Implementation Notes

### GF(2^128) Alpha Multiplication
```verilog
function [127:0] gf_mul_alpha;
    input [127:0] a;
    reg [127:0] result;
    begin
        result = {a[126:0], 1'b0};  // Shift left
        if (a[127])  // If MSB was 1, reduce
            result = result ^ 128'h87;
        gf_mul_alpha = result;
    end
endfunction
```

### Optimization Option
对于高性能场景，可以使用查找表或并行计算 alpha^N。
